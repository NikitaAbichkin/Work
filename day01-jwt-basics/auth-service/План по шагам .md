# План реализации ТЗ (Цели, Задачи, Результаты, AI-планирование)

Данный план описывает пошаговую реализацию функционала без написания самого кода, от уровня базы данных до API контроллеров.

---

## Текущее состояние проекта (что уже сделано)
- ✅ Goals CRUD — базовые эндпоинты POST/GET/PATCH/DELETE реализованы
- ✅ Таблица `stages` = `tasks` из ТЗ (решение: не создавать отдельную таблицу, а использовать stages)
- ✅ URL эндпоинтов исправлены на `/tasks`
- ✅ `estimated_time` (String) → `estimated_minutes` (Integer) — исправлено
- ✅ `updated_at` добавлен в Stage с `@PreUpdate`
- ✅ Результаты задачи хранятся в Stage (`result_text` + `result_images`) — отдельная сущность Result не нужна
- ❌ Goals: нет правильных 403/404 ответов (бросается `IllegalArgumentException` вместо нужных исключений)
- ❌ Фильтрация и сортировка целей — не реализованы
- ❌ AI-планирование — только заглушка по неверному URL (`/ai-decompose` вместо `/api/v1/ai/plan`)
- ⚠️ БАГ: `CreateGoalRequest.getStagesFromDto()` вызывает `stage.setEstimatedTime()` — метод больше не существует, нужен `setEstimatedMinutes()`

## Шаг 6: Разработка контроллеров (REST API) ⚠️
1. ⚠️ **`GoalController`** — есть, но:
   - ✅ POST `/api/v1/goals` → 201
   - ✅ GET `/api/v1/goals` → 200 (с пагинацией)
   - ✅ GET `/api/v1/goals/{id}` → 200
   - ✅ PATCH `/api/v1/goals/{id}` → 200
   - ✅ DELETE `/api/v1/goals/{id}` → 204
   - ❌ GET `/api/v1/goals` — нет query-параметров `status`, `sort`, `order`
2. ⚠️ **Tasks эндпоинты** (в GoalController):
   - ✅ POST `/{goalId}/tasks` → 201
   - ✅ PUT `/{goalId}/tasks/{stageId}` → 200
   - ⚠️ DELETE `/{goalId}/tasks/{stageId}` → возвращает 200 вместо 204
   - ❌ GET `/{goalId}/tasks` — нет (список задач цели)
   - ❌ GET `/{goalId}/tasks/{id}` — нет (задача по ID)
3. ~~`ResultController`~~ — не нужен
4. ❌ **`AiController`** — не реализован (есть только заглушка `/ai-decompose`)
