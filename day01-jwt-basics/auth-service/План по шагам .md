## Шаг 6: Разработка контроллеров (REST API) ⚠️
1. ⚠️ **`GoalController`** — есть, но:
   - ✅ POST `/api/v1/goals` → 201
   - ✅ GET `/api/v1/goals` → 200 (с пагинацией)
   - ✅ GET `/api/v1/goals/{id}` → 200
   - ✅ PATCH `/api/v1/goals/{id}` → 200
   - ✅ DELETE `/api/v1/goals/{id}` → 204
   - ✅ GET `/api/v1/goals/searchForParameters` — фильтрация по `status`, `priority` с пагинацией
   - 🐛 **БАГ**: `@GetMapping("{stageId}")` (строка 209) **конфликтует** с `@GetMapping("/{id}")` (строка 79) — Spring не различит два GET с одинаковым паттерном `/{param}`. Эндпоинт для одного stage должен быть `GET /{goalId}/tasks/{stageId}`
2. ⚠️ **Tasks эндпоинты** (в GoalController):
   - ✅ POST `/{goalId}/tasks` → 201
   - ✅ PUT `/{goalId}/tasks/{stageId}` → 200
   - ⚠️ DELETE `/{goalId}/tasks/{stageId}` → возвращает 200 вместо 204
   - ✅ GET `/{goalId}/stages` — список задач цели (с пагинацией, проверкой владельца)
   - ⚠️ **Замечание**: путь `/stages` не согласован с остальными эндпоинтами, которые используют `/tasks`
   - ❌ GET `/{goalId}/tasks/{id}` — нет (одна задача по ID). Вместо этого `GET /{stageId}` конфликтует с Goal
3. ~~`ResultController`~~ — не нужен
