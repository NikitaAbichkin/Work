# Технический контекст проекта GoalWays (Backend) для передачи

Этот документ предназначен для другой AI-модели, чтобы она могла мгновенно понять логику, структуру и текущий прогресс проекта без необходимости глубокого сканирования.

## 1. О проекте
**GoalWays** — платформа для постановки целей с элементами геймификации. Бэкенд строится на Spring Boot. Текущая фаза: Реализация надежного модуля авторизации (ТЗ1) и подготовка к CRUD целей.

## 2. Архитектурная логика (Как всё работает сейчас)
*   **Регистрация**: `POST /api/register`. Принимает `username`, `password`, `email`. Пароль хешируется BCrypt (cost 12). Создается запись в БД со статусом `NOT_CONFIRMED`.
*   **Верификация**: Генерируется 4-значный цифровой код (хранится в таблице `confirmation_codes`, связан с `User`). Код отправляется на почту через `ResendEmailService`.
*   **Подтверждение**: Предусмотрен метод `AuthService.confirm()`, который сверяет код, меняет статус юзера на `ACTIVE` и удаляет код. Эндпоинт в контроллере пока пуст.
*   **Логин**: Проверяет пароль и обязательно статус (`ACTIVE`). Если статус `NOT_CONFIRMED`, логин запрещен. Выдает JWT Access Token.
*   **Формат ответов**: В проекте есть `dto/ApiResponse.java`. Цель — чтобы ВСЕ ответы были в формате `{"status": "success", "data": {...}}`. Сейчас это внедрено лишь частично.

## 3. Текущий стек
*   Java 21, Spring Boot 3.4
*   PostgreSQL + Flyway (Миграции в `src/main/resources/db/migration`)
*   Spring Security + JWT
*   Resend API для почты (ключ в `application.yml`)
