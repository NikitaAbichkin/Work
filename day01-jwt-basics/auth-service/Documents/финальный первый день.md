# Контекст для завершения ДНЯ 1 (Auth Service)

Этот документ содержит текущий статус проекта и конкретные задачи для завершения первого этапа разработки.

## 1. Текущий стек и состояние
- **Стек:** Java 21, Spring Boot 3.x, PostgreSQL, Flyway, Spring Security (JWT), Resend API (Email).
- **Регистрация:** Работает. Пароли хешируются BCrypt (cost 12). Создается запись в `users` (статус `NOT_CONFIRMED`) и `confirmation_codes`.
- **Email:** Настроена отправка 4-значных кодов через `ResendEmailService`.
- **Авторизация:** Метод `login` проверяет пароль и статус пользователя, выдает JWT (Access Token).

## 2. Что УЖЕ сделано (95-100%)
- Миграции БД (V1-V3) для пользователей и кодов подтверждения.
- Модели `User` и `ConfirmationCode`.
- `AuthService.register()` — полная логика создания юзера и кода.
- `AuthService.login()` — базовая проверка и выдача токена.
- `ApiResponse.java` — DTO обертка (создана, но не внедрена).

## 3. ОШИБКИ И ПРОПУСКИ (Нужно исправить сейчас)
1. **Пакеты:** `GlobalExceptionHandler` лежит в `src/main/java/exception`. Нужно перенести в `com.example.auth.exception`, чтобы Spring `@RestControllerAdvice` заработал.
2. **Контроллер (`AuthController.java`):**
   - Базовый путь `/api`, нужно сменить на `/api/auth`.
   - Эндпоинт `confirm` — пустая заглушка.
   - Эндпоинт `resend-confirmation` — отсутствует.
   - Ответы методов `register` и `login` возвращают `Map` или голые DTO, а должны быть обернуты в `ApiResponse`.
3. **Refresh Token:** Полностью отсутствует. Нужно создать таблицу `refresh_tokens`, сущность, репозиторий и логику генерации при логине.
4. **JWT:** В токен нужно добавить Claim `user_id`.

## 4. Конкретные задачи для следующей итерации:

### Задача А: Рефакторинг эндпоинтов
- Обновить `AuthController` на `@RequestMapping("/api/auth")`.
- Реализовать `GET /confirm?username=...&code=...`.
- Реализовать `POST /resend-confirmation` (с логикой проверки email и генерации нового кода).
- Обернуть все ответы в `ResponseEntity<ApiResponse<T>>`.

### Задача Б: Реализация Refresh Token
- Создать миграцию `V4__add_refresh_tokens.sql` (id, token, user_id, expiry_date).
- Создать модель `RefreshToken` и `RefreshTokenRepository`.
- Обновить `AuthService`: при логине генерировать и Access, и Refresh токены.
- Создать эндпоинт `POST /api/auth/refresh`.

### Задача В: Безопасность
- Обновить `SecurityConfig`, чтобы разрешить доступ к `/api/auth/**`.
- Обновить `JwtService`, чтобы включать `user_id` в payload токена.
