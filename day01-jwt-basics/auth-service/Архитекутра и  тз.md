## 1. Авторизация

Все запросы к защищённым эндпоинтам требуют JWT токен в заголовке:

```
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...
```

### Открытые эндпоинты (без токена):
| Эндпоинт | Описание |
|---|---|
| `POST /api/auth/register` | Регистрация |
| `POST /api/auth/login` | Вход |
| `POST /api/auth/forgot-password` | Запрос сброса пароля |
| `POST /api/auth/reset-password` | Сброс пароля по токену |
| `GET /swagger-ui/**` | Swagger документация |

### Все остальные эндпоинты — закрытые (требуют Bearer токен).

### Токены:
| Тип | Время жизни | Где хранить (фронт) |
|---|---|---|
| `access_token` | 30 минут | Memory / Zustand store |
| `refresh_token` | 30 дней | HttpOnly cookie или AsyncStorage |

## 3. Обработка ошибок

### Формат ошибки:

```json
{
  "status": "error",
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Описание ошибки для пользователя",
    "details": { ... }
  }
}
```

### Таблица HTTP кодов и error codes:

| HTTP | Error Code | Когда |
|---|---|---|
| `400` | `VALIDATION_ERROR` | Невалидные данные (пустое поле, короткий пароль) |
| `400` | `BAD_REQUEST` | Некорректный запрос |
| `401` | `UNAUTHORIZED` | Нет токена или токен невалидный |
| `401` | `TOKEN_EXPIRED` | Токен истёк — фронт должен вызвать refresh |
| `403` | `FORBIDDEN` | Нет прав (чужая цель, чужой аккаунт) |
| `404` | `NOT_FOUND` | Ресурс не найден (цель, пользователь) |
| `409` | `CONFLICT` | Дубликат (username уже занят) |
| `413` | `FILE_TOO_LARGE` | Файл превышает лимит (10 MB) |
| `429` | `RATE_LIMIT` | Слишком много запросов |
| `500` | `INTERNAL_ERROR` | Серверная ошибка |

### Пример ошибки валидации:

```json
{
  "status": "error",
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Ошибка валидации",
    "details": {
      "username": "Минимум 3 символа",
      "password": "Минимум 8 символов"
    }
  }
}
```

### Пример ошибки 401:

```json
{
  "status": "error",
  "error": {
    "code": "TOKEN_EXPIRED",
    "message": "Токен истёк. Обновите токен через /auth/refresh"
  }
}
```

### 4.1 Auth

#### `POST /api/auth/register` — Регистрация

```
Доступ: публичный
```

Request:
```json
{
  "username": "nikita",
  "email": "nikita@mail.ru",
  "password": "mypassword123"
}
```

Response `201`:
```json
{
  "status": "success",
  "data": {
    "user_id": 1,
    "username": "nikita",
    "message": "Пользователь успешно зарегистрирован"
  }
}
```

Ошибки:
| HTTP | Когда |
|---|---|
| `409` | Username или email уже занят |
| `400` | Пароль < 8 символов, пустые поля |

---

#### `POST /api/auth/login` — Вход

```
Доступ: публичный
```

Request:
```json
{
  "username": "nikita",
  "password": "mypassword123"
}
```

Response `200`:
```json
{
  "status": "success",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiJ9...",
    "refresh_token": "dGhpcyBpcyByZWZyZXNo...",
    "token_type": "bearer",
    "expires_in": 1800
  }
}
```

Ошибки:
| HTTP | Когда |
|---|---|
| `401` | Неверный username или пароль |

---

#### `POST /api/goals/{id}/ai-decompose` — AI-разбивка цели на этапы

```
Доступ: авторизованный (только свои)
Асинхронный: да (DeepSeek API)
```

Response `202`:
```json
{
  "status": "success",
  "message": "AI анализирует цель. Этапы появятся в течение минуты"
}
```

> Бэкенд асинхронно (@Async) отправляет запрос к DeepSeek API,
> парсит ответ и создаёт stages для цели.
> Фронт может периодически дёргать `GET /api/goals/{id}`
> или получить push-уведомление когда готово.

## 5. Модели данных

### User
| Поле | Тип | Описание |
|---|---|---|
| `id` | Long | PK, автогенерация |
| `username` | String | Уникальный, не null |
| `email` | String | Уникальный, не null |
| `hashed_password` | String | BCrypt хеш |
| `is_active` | Boolean | Soft delete флаг |
| `created_at` | Timestamp | Дата регистрации |

### Goal
| Поле | Тип | Описание |
|---|---|---|
| `id` | Long | PK |
| `user_id` | Long | FK → User |
| `title` | String | Название цели |
| `description` | String | Описание (nullable) |
| `progress` | Integer | 0-100, авто-расчёт |
| `deadline` | Date | Дедлайн (nullable) |
| `is_archived` | Boolean | Флаг архива |
| `created_at` | Timestamp | |
| `updated_at` | Timestamp | |

### GoalStage
| Поле | Тип | Описание |
|---|---|---|
| `id` | Long | PK |
| `goal_id` | Long | FK → Goal |
| `title` | String | Название этапа |
| `is_completed` | Boolean | Выполнен ли |
| `order` | Integer | Порядок отображения |

### GoalAttachment
| Поле | Тип | Описание |
|---|---|---|
| `id` | Long | PK |
| `goal_id` | Long | FK → Goal |
| `filename` | String | Имя файла |
| `storage_path` | String | Путь в S3/MinIO |
| `size` | Long | Размер в байтах |
| `uploaded_at` | Timestamp | |

### GoalReminder
| Поле | Тип | Описание |
|---|---|---|
| `id` | Long | PK |
| `goal_id` | Long | FK → Goal |
| `frequency` | String | daily / weekly / custom |
| `time` | Time | Время напоминания |
| `is_enabled` | Boolean | Активно ли |

### Achievement
| Поле | Тип | Описание |
|---|---|---|
| `id` | Long | PK |
| `title` | String | Название |
| `description` | String | Описание |
| `icon` | String | Иконка |
| `condition_type` | String | Тип условия (goals_created, goals_completed, stages_completed) |
| `condition_value` | Integer | Сколько нужно для получения |

### UserAchievement
| Поле | Тип | Описание |
|---|---|---|
| `id` | Long | PK |
| `user_id` | Long | FK → User |
| `achievement_id` | Long | FK → Achievement |
| `progress` | Integer | 0-100 |
| `is_unlocked` | Boolean | Получено ли |
| `unlocked_at` | Timestamp | Когда получено |

### Notification
| Поле | Тип | Описание |
|---|---|---|
| `id` | Long | PK |
